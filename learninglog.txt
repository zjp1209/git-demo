#定义什么是git
git 是一个文件仓库  可以私有 也可以推送到公共区间github上 让别人拉取 

#基础命令
git init  #创建一个新的仓库
git add demo.html #把创建的这个文件 加入到暂存区 
 #三个区  工作区（git软件正在编辑的） 暂存区（add添加到类似购物车） 私有仓库区（类似已经付款买完的）
 
git commit -m "第一次提交" # -m 是简化了一个确认步骤 不然会调入道vim编辑器里去确认   “内容” 是对这个动作的解释说明
git log #查看日志 
# 我发现日志里面没有  原来是由于我第一次创建 要把自己的邮箱和用户名输入进去  才可以保存 这样方便追踪
git config --global user.name "名字“
git config --global user.email "邮箱”   
#之后再提交一遍 就能查看日志  就有了 或这查看状态  如果是红色的 就是没有被暂存的 
 绿色就是被暂存的 而啥也查不到可能是 也许就是加入成功了

#忘记的内容回顾
git mv “原文件名” “新文件名” #改文件名
#修改文件名的时候  要用英文 不然改名过程的描述会显示乱码 再一个文件之间不要有空格
不然就会让系统误认为是两个参数 解决方式是 要么去掉空格 要么加上\ 来连接后面的内容
比如 name big.txt 要是下命令 最好改为 name\ big.txt  这样系统就会把它当成一个
整体来看待  不然就直接改成 namebig.txt
#当我在mv 改名的时候 弄错了结尾写错成了中文双引号 导致系统认为是内容的一部分而出现
等待输入的状态而出现>这个符号 所以需要按 ctrl c 来中断请求 重新输入
